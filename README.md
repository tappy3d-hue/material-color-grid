# Material Color Grid Texture

選択した複数オブジェクトのマテリアルのBase Colorを **1枚の共有グリッドテクスチャ** にまとめ、共有マテリアルとして全オブジェクトに割り当てるBlenderアドオンです。メッシュは結合されず分割されたままで、元のマテリアル割り当ては頂点グループとして保存されます。

ベースカラーに加えてラフネス／メタリックのテクスチャ生成、色の追加（インクリメンタル更新）、グリッドからのマテリアル復元、未使用色の掃除、極小テクスチャ化、PNG書き出しに対応。主にBlender → Roblox Studio 移行で、テクスチャアセットの枚数・容量を最小化する用途を想定しています。

![grid example](docs/example.png)

## 主な機能

- 選択オブジェクト全体から Base Color を集約して1枚のグリッドテクスチャを生成
- ラフネス／メタリックもグレースケールのグリッドテクスチャとして生成（値は0.1刻みで四捨五入）
- 柄入りテクスチャのアトラス化（Base Colorに画像が繋がったマテリアルは、その柄をセルに焼き込み）
- オブジェクト単位の処理モード（Atlas / Vertex Color / JSON / None）
- 頂点カラー焼き込みモード（FBX/glTFでRobloxへ）
- JSON出力モード（マテリアルごとにパーツ分割し、Roblox側で動的着色。専用プラグイン同梱）
- Roblox向け検証パネル（トライアングル数・未適用スケール・UV範囲・テクスチャ解像度などをチェック）
- Bake後の自動PNG書き出し＋参照化（FBXにテクスチャを埋め込みやすくする）
- 後から別オブジェクトを追加しても、既存色を保持したまま新色を追加して更新
- グリッドから元の色ごとのマテリアルを復元（焼き戻しの逆変換）
- テクスチャ／マテリアルの命名・リネーム
- グリッドを使う全オブジェクトを一括選択
- 未使用色の掃除（Compact）でグリッドを詰める
- 最小解像度モードで極小テクスチャ化
- PNG書き出し（Robloxアップロード用）
- 元のマテリアル名で頂点グループを作成し、後から面を再選択可能
- アドオン内アップデート（プリファレンスからGitHubの最新版をチェック＆インストール）

## インストール

1. `material_color_grid.py` をダウンロード（ファイル名にドットを入れないこと）
2. Blender で `編集 → プリファレンス → アドオン → インストール...`
3. `.py` ファイルを選んで、チェックボックスを有効化

Blender 3.x / 4.x で動作確認しています。

## UI

有効化すると、3Dビューポートのサイドバー（`N` キーで開閉）に **「Color Grid」タブ** が追加されます。各操作は `F3` 検索や `オブジェクト` メニューからも実行できます。

### Bake / Update セクション
- **Grid Name** — 新規グリッドのテクスチャ名・マテリアル名（リネーム時の目標名にも使用）
- **Minimal Resolution** — ONで極小テクスチャ化（`Cell Pixels` が表示される）
- **Resolution / Cell Pixels** — 通常解像度、または最小モード時のセルあたりピクセル数
- **Roughness Map / Metallic Map** — ラフネス／メタリックのグレースケールテクスチャも生成（値は0.1刻みで四捨五入）
- **Vertex Groups / Remap UVs / Replace Slots / Update All Users** — オプション
- **Bake Selected to Grid** — 上記設定でグリッドに焼く／更新

### Tools セクション
- **Rename Current Grid** — 選択中のグリッドのテクスチャ＆マテリアルを Grid Name にリネーム
- **Select Objects Using Grid** — そのグリッドを使う全オブジェクトを一括選択
- **Compact Grid (Remove Unused)** — どのオブジェクトにも使われていない色を除去して詰める
- **Export Textures (PNG)** — テクスチャをPNGとして書き出し。ダイアログの **Reference Exported File (Unpack)**（デフォルトON）で、書き出したPNGを画像の参照先に設定しパック解除する。これによりFBXエクスポート時にテクスチャが実ファイルとして参照され、Roblox Studioでテクスチャが読み込まれない問題を回避できる。エクスポートはグリッドごとの固定名（シード入り）に上書き保存するため、同じグリッドの出し直しは同じPNGを更新し、別グリッド・別プロジェクトのファイルとは衝突しません

### Reverse セクション
- **Restore Materials** — グリッドから個別マテリアルを復元

## 使い方

### 1. グリッドに焼く（初回）

1. 色をまとめたいメッシュオブジェクトを複数選択
2. Grid Name を入力し、**Bake Selected to Grid**

全オブジェクトをまたいでユニークな色だけが集約され、`ceil(√n) × ceil(n/cols)` のグリッドテクスチャ（例：4色 → 2×2、10色 → 4×3で2マス空き）が1枚生成され、共有マテリアルが割り当てられます。新規グリッドの名前には自動でランダムシードが付与され（例 `ColorGrid_a8f3c1`）、プロジェクトをまたいでもファイル名が衝突しません。**ジオメトリは結合されず、メッシュは分割されたままです。**

### 2. 色を追加して更新（2回目以降）

すでにグリッドが適用されたオブジェクトと、まだ元のマテリアルを持つオブジェクトを **一緒に選択** して **Bake** すると：

1. 既存の共有マテリアルに保存されたマニフェスト（色の台帳）から既存色をすべて保持
2. 新オブジェクトのBase Colorを末尾に追加
3. テクスチャを作り直し
4. 既存オブジェクトのUVを新レイアウトに自動で貼り直し（色は維持）

**Update All Objects Using Texture**（デフォルトON）で、選択していない既存オブジェクトのUVも自動で追従させ、色崩れを防ぎます。事前に **Select Objects Using Grid** で対象を一括選択しておくのも有効です。

### 3. マテリアルを復元する（逆変換）

グリッドが適用されたオブジェクトを選択して **Restore Materials** を実行すると、各面のUVから色セルを判定し、色ごとのマテリアル（Principled BSDF）を再生成して割り当て直します。個別マテリアル方式（JSON移行向けなど）に戻せます。

### 4. 整理して書き出す（Roblox向け）

1. 不要になった色は **Compact Grid** で除去してグリッドを詰める
2. **Minimal Resolution** をONにして再 **Bake** → 極小テクスチャ化（`Cell Pixels=8` 程度ならRobloxのフィルタでも色のにじみが出にくい）
3. **Export Textures (PNG)** でPNGを書き出す（**Reference Exported File** をONにしておくと画像がパック解除され、ディスク上の実ファイルを参照する状態になる）
4. FBXエクスポート（Path Mode = `Copy` ＋ Embed Textures 推奨）→ Roblox Studio でインポート

> パック状態（.blend埋め込み）のままFBX書き出しすると、テクスチャがディスク上の実ファイルとして参照されず、Roblox側で読み込まれないことがあります。**Reference Exported File** で実ファイル参照にしておくと回避できます。Robloxの埋め込みテクスチャ読み込みは不安定なこともあるため、書き出したPNGを別途アップロードしてTextureID/SurfaceAppearanceで手動適用するのが最も確実です。

## 補足

- 新規グリッドの名前は `<Grid Name>_<ランダムシード>` 形式になります（例 `ColorGrid_a8f3c1`）。Grid Name を空にするとシードのみ。これによりプロジェクトをまたいでも名前・ファイル名が衝突しません。更新時は名前を変えず、選択オブジェクトが使うテクスチャを再利用します。既存グリッドの名前は **Rename Current Grid** で変更できます。
- マニフェストは共有マテリアルのカスタムプロパティ `mcg_manifest` として .blend に保存され、ファイルを開き直しても引き継がれます。
- 色の並び順（インデックス）は維持され、新色は末尾に追加されるので既存色は新レイアウトに正しく追従します。
- 同名マテリアルが既にマニフェストにある場合は、セルを増やさずその色を最新値に更新します。
- Principled BSDFが無いマテリアルはビューポートのdiffuse colorにフォールバックします。
- リンク複製（同じメッシュデータを共有するオブジェクト）は二重処理されないようガードしています。
- 色は linear → sRGB エンコードして書き込むので、レンダー結果が元のBase Colorと一致します。

## 処理モード（オブジェクト単位）

サイドバー上部「Objects & Modes」で、選択中の各オブジェクトに処理モードを割り当てます（オブジェクトに保存され、未設定なら自動判定）。

- **Atlas** — マテリアル色／テクスチャを共有グリッドテクスチャに焼き込む
- **Vertex Color** — Base Colorを頂点カラーに焼く（FBX/glTFでRobloxへ。テクスチャ不要）
- **JSON** — 色を焼かず、マテリアルごとにパーツ分割してJSON出力（Roblox側で動的着色）
- **None** — 何もしない

「Bake / Process」の共通ボタンが、各オブジェクトのモードに従って Atlas・Vertex をまとめて処理します。JSONは「Export JSON…」ボタンから（実行前に分割内容を確認するダイアログが出ます）。

### JSONモードとRobloxプラグイン

JSONモードは「1パーツ＝1色」のRoblox前提に合わせ、複数マテリアルのオブジェクトをマテリアルごとに分割します（グリッド化済みは元マテリアルに復元してから分割）。出力したJSONを同梱の `ColorImporter.rbxmx`（Studioプラグイン）に貼り付け、対象モデルを選択して適用します。キーは `<名前>_<マテリアル名>_<ID>` で名前衝突を防ぎ、適用後は各パーツに `MCGColor` 属性を書き戻すので再適用も安定します。

## Roblox Check（検証）

サイドバーの「Roblox Check」で **Run Roblox Check** を押すと、選択中オブジェクトを検査して警告を一覧表示します（トライアングル数の上限、スケール未適用、UVマップの有無・0-1範囲、テクスチャ解像度の過大、複数マテリアルでAtlas未設定など）。閾値（Max Triangles / Max Texture）はパネルで調整できます。

## FBX向け自動エクスポート

Atlas設定の **Auto Export + Unpack After Bake** をONにすると、Bake後にアトラスPNGを.blendと同じフォルダへ保存し、画像をその実ファイル参照に切り替えます（パック解除）。これによりFBXエクスポート（Path Mode = Copy ＋ Embed Textures）でテクスチャが埋め込まれます。.blendを保存してから使ってください。

## アップデート

`編集 → プリファレンス → アドオン → Material Color Grid Texture` を展開すると、更新用のUIがあります。

- **Check for Updates** — GitHubの最新リリースと現在のバージョンを比較
- 新しいバージョンがあれば **Download & Install Update** が表示される → 押すと最新の `material_color_grid.py` をダウンロードしてアドオンを上書き
- インストール後、**Blenderを再起動**すると反映されます
- **Open Releases Page** でリリースページをブラウザで開けます

> この更新機能自体は、その機能を含むバージョン以降でのみ使えます。最初の導入時は手動でインストールしてください。
